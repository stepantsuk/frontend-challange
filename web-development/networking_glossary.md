**Connection** - фрагменты связанной информации, которые передаются по сети. Как правило, соединение устанавливается до передачи данных (путем следования процедурам, изложенным в протоколе) и может быть разорвано после окончания передачи данных.
**Или**
**Сетевое соединение** - это логический канал связи, созданный между двумя устройствами для обмена данными. Обычно соединение предварительно устанавливается (согласно правилам протокола), затем происходит передача данных, после чего соединение завершается.
---
**Packet / Пакет** - это наименьшая единица данных, целенаправленно передаваемая по сети. При сетевом взаимодействии пакеты служат конвертами, которые переносят данные от начальной к конечной точки

*Пакет содержит:*
- заголовок, в котором служебная информация, IP-адреса источника и назначения
- метки времени
- количество сетевых переходов / данные о маршрутизации
---
**Network Interface / Сетевой интерфейс** - это программный интерфейс для взаимодействия с сетевым оборудованием. Это точка подключения устройства к сети. Физически это может быть сетевая карта, а программно - ее представление в операционной системе (например, Ethernet-адаптер или подключение Wi-Fi). У одного устройства может быть несколько сетевых интерфейсов, каждым из которых можно управлять независимо 

---
**LAN** - локальная вычислительная сеть, ограниченная сравнительно небольшой областью/территорией действия: дом, офис или здание. Такие сети являются частными и могут быть как изолированы от публичного доступа в интернет, так и иметь к нему подключение через маршрутизатор.

**WAN** - глобальная вычислительная сеть. Термин применим для сетей, которые гораздо обширнее локальных. Термин подразумевает географически распределенную сеть, которая может объединять несколько локальных сетей. Обычно под WAN подразумевают интернет.

---
**Protocol / Протокол** - это набор правил и соглашений, который определяет, как устройства и программы в сети обмениваются информацией. Он стандартизирует процесс, чтобы устройства разных производителей могли «понимать» друг друга, определяя формат данных, порядок их передачи, методы обнаружения и исправления ошибок. 

*Функции протоколов:*
- Стандартизация - возможность взаимодействия между разными устройствами и программами, используя общие правила 
- Надежность - включают механизмы проверки данных и исправления ошибок во время передачи
- Адресация - обеспечение точности определения отправителя и получателя
- Маршрутизация - определение оптимального пути для передачи данных по сети
- Управление потоком - регулируют скорость передачи данных, чтобы избежать перегрузок

Сетевое взаимодействие работает по принципу инкапсуляции, когда несколько различных протоколов работают друг поверх друга или иными словами по принципу стека, где каждый вышележащий протокол использует услуги нижележащего, добавляя свою функциональность.

У протоколов присутствует иерархичность, которая начинается с с базовых протоклов физического уровня и продолжается до сложных прикладных протоколов.

_**Виды протоколов**_:
**MAC-адресация** - это система уникальной аппаратной идентификации, сетевых устройств.
- уникальный идентификатор - его получает каждое устройство при производстве
- аппаратная адресация - позволяет однозначно идентифицировать устройство, независимо от изменений в его программном обеспечении
- практическая значимость - несмотря на принадлежность к низкому канальному уровню, МАС-адресация используется постоянно

**IP** - это один из фундаментальных протоколов, обеспечивающих работу интернета. IP-адреса являются уникальными в каждой сети и позволяют машинам обращаться к друг другу через сеть. Он реализован на сетевом уровне модели TCP/IP.
Сети объединяются и на границе сетей должна быть маршрутизация. Этот протокол подразумевает работу в ненадежной сети и подразумевает наличие нескольких путей к одному и тому же месту назначения, между которыми возможно переключаться динамически.
_Особенности_:
- уникальная адресация - каждое устройство в сети имеет уникальный IP-адрес для идентификации
- межсетевое взаимодействие - обеспечивает передачу данных между различными сетями через маршрутизаторы
- отказоустойчивость - работает в условиях нестабильных сетей с динамической маршрутизацией


Наиболее распространенные виды реализации протоколов IPv4(формат: 192.168.1.1) и IPv6(формат: 2001:db8::ff00:42:8329).

**ICMP** - Internet Control Message Protocol - это служебный протокол для обмена диагностической и управляющей информацией между сетевыми устройствами. Он используется для отправки сообщений между устройствами для индикации их доступности или ошибок. Эти пакеты применяются в различных инструментах сетевой диагностики, таких как ping или traceroute.

Принцип работы: ICMP-сообщения генерируются автоматически при возникновении проблем с доставкой обычных пакетов данных, выполняя роль системы обратной связи в IP-сетях.

Основные функции:
- передача уведомлений об ошибках
- проверка доступности узлов с помощью утилиты ping
- определение маршрута следования пакетов через traceroute

**UDP** - User Datagram Protocol - это транспортный протокол без установления соединения, подтверждения получения данных, обеспечивающий минимальную задержку передачи данных. (Протокол жертвует надежностью в пользу скорости)

Ключевые особенности:
- ненадежная доставка, тк не требуется подтверждения получения пакетов
- высокая скорость, тк отсутствуют затраты времени на установку соединения и повторную передачу данных
- не отслеживается состояние соединения между передачами

Области применения:
- VoIP-телефония и видеокоференции
- многопользовательские онлайн-игры
- потоковое видео и аудио
- DNS запросы и другие кратковременные взаимодействия


**HTTP** - протокол передачи гипертекста. Это протокол прикладного уровня, фундаментальный для работы веба. HTTP определяет формат запросов и ответов между клиентами (браузерами) и серверами.
Ключевые методы HTTP
GET - получение данных с сервера
POST - отправка данных на сервер
DELETE - удаление данных на сервере

---
**Port/Порт** - Это числовой идентификатор, который помогает компьютеру определять какому именно приложению или службе предназначены входящие данные и какое приложение их отправляет.

---
**Firewall** - это система фильтрации сетевого трафика, которая защищает сервер, контролируя все входящие и исходящие соединения. Он работает на основе набора правил, которые разрешают или запрещают передачу данных через определенные порты. 

---
**NAT (Network Address Translation)** - это технология, позволяющая нескольким устройствам в локальной сети использовать один общий публичный IP-адрес для выхода в интернет. Маршрутизатор, используя NAT, "подменяет" частные IP-адреса внутренних устройств на свой публичный адрес при отправке запросов наружу и корректно распределяет входящие ответы по нужным устройствам внутри сети. Это решает проблему нехватки IPv4-адресов и повышает безопасность.

---
**VPN** - это технология, создающая зашифрованный защищенный "туннель" через публичную сеть. Она позволяет удаленным пользователям или филиалам получить доступ к частной локальной сети, как если бы они были физически подключены к ней напрямую. 

---
**Network Layers**

Сеть необходимо представлять не только как взаимодействие между узлами в горизонтальной плоскости, но и как вертикальную, слоистую структуру внутри каждого конкретного компьютера или сети.

Сетевое взаимодействие реализовано в виде стека протоколов, где каждый уровень решает свою конкретную задачу.

1. Абстракция: верхние уровни работают с логическими уровнями данных, не вдаваясь в детали их физической передачи нижними уровнями (например, кабелем или радиосигналом).
2. Многоразовость: высокоуровневые приложения могут использовать одни и те же нижние уровни без необходимости их переписывания.
3. Инкапсуляция: при отправке данные движутся вниз по стеку и каждый уровень добавляет к ним свой заголовок (обертку). При получении данные движутся вверх по стеку и каждый уровень считывает свой заголовок, удаляет его и передает распакованные данные уровню выше. Классические модели, описывающие эти слои - это OSI и TCP/IP

---
**TCP/IP Model** (transmission control protocol / internet protocol)- это сетевая модель передачи данных.
Набор интернет протоколов обеспечивает сквозную передачу данных, определяющую, как данные должны пакетироваться, обрабатываться, передаваться, маршрутизироваться и приниматься.
В модели предусматривается прохождение информации через четыре уровня:
1. Прикладной (Application) - уровень приложений и сервисов. Его задача - представить данные пользователю и подготовить их к передаче через интерфейсы и АПИ.
2. Транспортный уровень (Transport) - уровень контроля связи. Он отвечает за надежную доставку данных между приложениями на разных хостах, используя порты для идентификации служб. Главные протоколы - TCP (надежный) и UDP (быстрый).
3. Сетевой уровень (Internet): Уровень маршрутизации. Его задача - доставить пакеты до нужного IP-адреса в глобальной сети, выбирая оптимальный путь. Основной протокол - IP (internet protocol)
4. Канальный уровень (Link): Уровень передачи данных по физической среде (например, по Ethernet, wifi, bluetooth). Данные делятся на небольшие фреймы и передаются между непосредственно соединенными устройствами, каждый фрейм так же содержит служебные данные. Используется МАС-адреса для передачи фреймов, это уникальные физические адреса устройств - по ним протоколы определяют отправителей и получателей.

---
**Интерфейсы** - это точки подключения компьютера к сети, соответствующие физическим или виртуальным сетевым устройствам.

Физические интерфейсы соответствуют сетевым картам и настраиваются отдельно.
Виртуальный интерфейс обеспечивает сетевое взаимодействие между программами в рамках одной системы.

---
**DNS** - domain name system - это распределенная система преобразования доменных имен в IP-адреса.

Это протокол прикладного уровня, который предоставляет удобный для человека механизм именования интернет-ресурсов. 

Основные функции:
- Трансляция удобных для человека доменных имен в машинные IP-адреса
- Организация иерархической структуры доменных имен
- Кеширование запросов для ускорения разрешения имен

---
**SSH** - secure shell - означает безопасную оболочку. Это зашифрованный протокол прикладного уровня используется для безопасного взаимодействия с удаленным сервером.

Ключевые особенности:
- сквозное шифрование
- аутентификация по ключам или паролям
- поддержка туннелирования и передачи файлов (SFTP)

Значение: благодаря наджености и универсальности, SSH стал основой для многих систем администрирования и инструментов разработки.
