# Domain Name System

Чтобы оказаться на сайте example.ru, пользователю необходимо ввести его в адресной строке браузера - это доменное имя сайта. А серверу, чтобы обработать запрос и перевести пользователя на сайт, нужен IP-адрес. Именно ДНС преобразует запросы сайтов в IP-адреса, понятные для серверов.

IP-адрес - это уникальный числовой идентификатор, который используется для идентификации устройства в сети.

**Принцип работы DNS**:
- Локальный кеш. Браузер смотрит в свой локальный кеш и кеш ОС. Если подходящая информация есть, то сразу открывается подходящая страница.
- Рекурсивный сервер. При отсутствии данных в кеше запрос направляется на рекурсивный DNS-сервер. Обычно он контролируется провайдером. Этот сервер выполняет роль посредника.
- Корневой DNS-сервер или Root Server. Корневой сервер предоставляет информацию о серверах, которые отвечают за домены верхнего уровня.
- TLD сервер. Рекурсивный сервер посылает запрос TLD серверу, он передает записи о домене второго уровня. Допустим, TLD сервер зоны .com передает данные о сайте spotify.com и перенаправляет их дальше, иными словами TLD сервер сообщает, какой авторитетный сервер хранит IP адрес для spotify.com. 
- Авторитетный сервер. Рекурсивный сервер связывается с сервером, который считается авторитетным для домена, хранящего информацию об определенном сайте и забирает IP-адрес.
- Кеширование результата. Рекурсивный сервер сохраняет связку данных в собственном кеше, что в дальнейшем ускоряет процесс подбора IP-адреса. Период хранения определяется параметром TTL в настройках DNS-сервера.

DNS построен по иерархическому принципу, который обеспечивает масштабируемость и надежность системы. 

**Виды DNS-записей**
- А - показывают IPv4-адрес;
- АААА - показывают IPv6-адрес;
- CNAME - отвечают за переадресацию между доменами;
- MX - определяют почтовые серверы, ответственные за прием электронной почты для домена;
- TXT - включают текст, полезный при верификации.
